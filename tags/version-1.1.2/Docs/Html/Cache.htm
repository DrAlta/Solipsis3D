Cache Description

The cache  has exactly the same fucntionality on the server and on teh peer (and navigator for rcs option, the peer is embedded)
The unique class handling the cache is the cache manager (Communication\RakNetObjects\CacheManager.cpp)
It is used by both server and client side.

Directory location 


At startup both modules search for a "Media\Cache" directory in the current working directory (by default the directory where reside the exe)
If not found it will search in parent directories with untill this directory is found.

If you use the server and the Peer/Navigator on the same computer please copy the Media/cache directory  into the Navigator directory.

Directory content

this directory should contain a cache.xml file referencing all resources files and its version number. If the file is not found a empty file is created (all already preset files are then ignored)
each file is a zipped ogre resource directory.
  • ssf for scenes
  • saf for avatars
  • sof for objects
rename it with the .zip extension to view its content.
It contains :
  • meshes (.mesh)
  • material script descriptions (.material),
  • textures (.jpg and .png),
  • scene description(.osm),
  • avatar skeleton(.skeleton)
  • and avatar nature (.nature)

see ogre forum and tutorial for more infos about these formats.

The server also use this directory to maintains scene content (avatar and objects status and positions) via the SceneID.xml files.
The Navigator client maintains the last avatar status via another xml file.

The ***_Ref.xml files are references files. Xml files are modified during a server session, to recover the initial state maunch the cleancache.bat. It will delete all xml file and recover the initial states using this ref files.
