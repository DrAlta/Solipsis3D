Solipsis Coding Rules

Some rules for developpers

  1. Because those directories are filled during the generation,
    I DO NOT commit anything into Common Dependencies
    (except for plugins because some people cannot build them without ATL on Visual Studio Express edition)

  2. if you add something into DevTools directory.
    Then copy libraries/includes/binaries into Dependencies directory with post-build events (VStudio or batch)

  3. if you generate 1 binary then copy all files this binary needs into Common/bin/<binary name>,
     idea is I can launch this binary from this directory without adding anything else ...
  4. Before creating 1 new directory check if you cannot use the dedicated directory :
    media files into Media, 
    3rd-parties into DevTools . . .

  5. Create as many configuration projects as you can (Visual Studio 2005 = vc8, Visual Studio 2008 = vc9, ...). 
    Visual Studio 2003 = vc7 is no more used

  6. if you need some third-parties (libraries, ...), check it is not already available into DevTools directory :
     tinyxml, pthreads, . . .

  7. if you need some third-parties and it is not yet into DevTools directory
     then try to choose 1 crossplatform version and add it into DevTools directory

  8. if you add something into DevTools directory.
    Then copy libraries/includes/binaries into Dependencies directory with post-build events (VStudio or batch)

  9. if you generate 1 binary then copy all files this binary needs into Common/bin/<binary name>,
     idea is I can launch this binary from this directory without adding anything else ...
  10. Of course you can't use space in file name, when intend to been fully unix compatible.

Coding rules

  1. IDE/editor config
     Tabs are replaced by spaces
     Tab = 4 spaces

  2. include <> if header file is outside of the project, include "" if header file is in the same project

  3. add/use doxygen comments

  4. always add current GPL licence header into your new source files

  5. all sources are in DOS format (for instance), check you commit files in this format (run a file cygwin command)
